Governance

Editorial, contribution, and versioning guidance for PEMFC ontology maintenance and release governance.

Editorial policy

  • Preserve stable public IRIs by default and avoid breaking local PEMFC terms without a migration note.
  • Prefer explicit annotation improvement, alignment, and modular separation over destructive refactoring.
  • Require review before turning weak lexical mapping suggestions into asserted equivalence.

Contribution path

  • Report ontology issues and mapping questions in the GitHub issue tracker.
  • Review mapping proposals and annotation drafts before importing them into the published schema.
  • Treat vocabulary growth, deprecation, and namespace changes as release-governed changes.

Versioning policy

  • Keep the ontology IRI stable while publishing version-specific releases and a latest alias.
  • Record prior versions and release metadata for every published release bundle.
  • Document deprecations and namespace migrations explicitly rather than silently rewriting identifiers.
