Internal FAIR signals are local release-readiness indicators. External OOPS! and FOOPS! results are reported separately so service outages do not masquerade as ontology quality.

FAIR Signals

Internal FAIR signals estimate release readiness from the built ontology package itself. They are conservative but not equivalent to live post-publication FAIR audits.

  • F / Findable GOOD 86 / 100
    Internal score for identifiers, versioning, citation, namespace, and findable publication metadata.
  • A / Accessible GOOD 85 / 100
    Internal score for machine-readable outputs, docs presence, resolver artifacts, and public access hooks.
  • I / Interoperable GOOD 90 / 100
    Internal score for reuse of imports, mappings, namespace hygiene, and standards-based serializations.
  • R / Reusable WATCH 65 / 100
    Internal score for license, provenance, versioning, definitions, examples, and release packaging.

ODK / ROBOT QC

ODK / ROBOT signals reflect standard ontology engineering QC and are reported in addition to H2KG-specific FAIR and publication checks.

  • ROBOT status GOOD completed
    ROBOT/ODK commands completed and actual ODK artefacts were collected.
  • ROBOT errors GOOD 0
    Shadow-mode ROBOT summary from the nested ODK workbench.
  • ROBOT warnings GOOD 0
    Warnings indicate shadow scaffolding or follow-up work still required before promotion.
  • Import refresh health GOOD 10 configured
    Imports are tracked through the nested ODK workbench and surfaced here alongside H2KG-specific checks.
  • Parity status WATCH minor differences under review
    ODK base artefact differs slightly from the current curated release and remains in shadow-mode comparison.
  • IRI drift ACTION detected
    Shadow mode should preserve existing H2KG identifiers.
  • HDO review coverage GOOD 10
    Local H2KG terms reviewed against HDO for data, metadata, identifier, schema, validation, and digital-object semantics.
  • HDO mapping coverage GOOD 130
    Mappings or reuse proposals that currently point to HDO anchors.
  • H2KG terms still local after HDO review WATCH 3
    Reviewed terms intentionally kept local until direct HDO reuse is accepted or a more precise HDO target is loaded.

Validation Signals

Validation signals summarize local structural, metadata, namespace, mapping, and optional SHACL checks on the release candidate.

  • Overall validation status GOOD pass
    Combined metadata, namespace, mapping, and SHACL validation checks.
  • SHACL conforms ACTION False
    Local SHACL shape execution when pyshacl is available.
  • Duplicate @id conflicts GOOD 0
    Duplicate source nodes are tolerated only when they merge without schema-type conflicts.
  • Missing labels GOOD 0
    Release-time missing labels on local schema terms.
  • Missing definitions GOOD 0
    Release-time missing definitions or comments on local schema terms.
  • Namespace violations GOOD 0
    Violations against the active namespace policy.
  • Mapping issues GOOD 0
    Mappings that remain risky or inconsistent after local checks.
  • OWL consistency hook WATCH skipped (owlready2 not installed)
    Optional OWL reasoner hook. It is skipped when owlready2 is not installed.
  • EMMO checks WATCH skipped (EMMOntoPy not installed)
    Optional EMMO convention hook. It is skipped when EMMOntoPy is not installed.
