Internal FAIR signals are local release-readiness indicators. External OOPS! and FOOPS! results are reported separately so service outages do not masquerade as ontology quality.

FAIR Signals

Internal FAIR signals estimate release readiness from the built ontology package itself. They are conservative but not equivalent to live post-publication FAIR audits.

  • F / Findable GOOD 86 / 100
    Internal score for identifiers, versioning, citation, namespace, and findable publication metadata.
  • A / Accessible GOOD 85 / 100
    Internal score for machine-readable outputs, docs presence, resolver artifacts, and public access hooks.
  • I / Interoperable GOOD 90 / 100
    Internal score for reuse of imports, mappings, namespace hygiene, and standards-based serializations.
  • R / Reusable WATCH 65 / 100
    Internal score for license, provenance, versioning, definitions, examples, and release packaging.

Transparency Hooks

External transparency hooks call public assessment services. Their availability depends on external infrastructure and network access.

  • OOPS! ontology pitfall scan WATCH service unavailable
    OOPS! something went wrong. There was an unexpected error.
  • FOOPS! FAIR assessment ACTION 46.7 / 100
    External FAIR-oriented ontology assessment against the release candidate. File mode does not assess accessibility.

FOOPS! Assessment

FOOPS! is an external FAIR-oriented ontology validator. In file mode it does not run accessibility checks, so the Accessible dimension may appear as not assessed.

  • Status GOOD assessed
    FOOPS! assessment completed in file mode. Accessible checks may remain unassessed.
  • Overall score WATCH 46.7 / 100
    Returned directly by the FOOPS! service.
  • F / Findable ACTION 60.0 / 100
    Returned directly by the FOOPS! service.
  • A / Accessible WATCH not assessed
    In file mode this dimension is commonly not assessed by FOOPS!.
  • I / Interoperable ACTION 33.3 / 100
    Returned directly by the FOOPS! service.
  • R / Reusable ACTION 44.4 / 100
    Returned directly by the FOOPS! service.
  • FOOPS! follow-up 1 WATCH F1
    Unexpected error while running the test: null
  • FOOPS! follow-up 2 WATCH F2
    The following metadata was not found: version iri
  • FOOPS! follow-up 3 WATCH R1
    The following metadata was not found: version info, citation
  • FOOPS! follow-up 4 WATCH R1
    The following metadata was not found: doi, logo, status, source, issued. Warning: The following OPTIONAL detailed metadata could not be found: previous version, backwards compatibility. Please consider adding them if appropriate.
  • FOOPS! follow-up 5 WATCH R1.2
    The following provenance information was not found: issued
  • FOOPS! follow-up 6 WATCH I2
    The ontology does not reuse vocabularies for common metadata
  • FOOPS! follow-up 7 WATCH I2

OOPS! Pitfalls

OOPS! is an external ontology pitfall scanner. Service errors are shown as availability problems rather than as zero pitfalls.

  • Status WATCH unavailable
    OOPS! something went wrong. There was an unexpected error.
  • Pitfall count WATCH not assessed
    Returned directly by the OOPS! service when the scan succeeds.

Validation Signals

Validation signals summarize local structural, metadata, namespace, mapping, and optional SHACL checks on the release candidate.

  • Overall validation status GOOD pass
    Combined metadata, namespace, mapping, and SHACL validation checks.
  • SHACL conforms ACTION False
    Local SHACL shape execution when pyshacl is available.
  • Missing labels GOOD 0
    Release-time missing labels on local schema terms.
  • Missing definitions GOOD 0
    Release-time missing definitions or comments on local schema terms.
  • Namespace violations GOOD 0
    Violations against the active namespace policy.
  • Mapping issues GOOD 0
    Mappings that remain risky or inconsistent after local checks.
  • OWL consistency hook WATCH skipped (owlready2 not installed)
    Optional OWL reasoner hook. It is skipped when owlready2 is not installed.
  • EMMO checks WATCH skipped (EMMOntoPy not installed)
    Optional EMMO convention hook. It is skipped when EMMOntoPy is not installed.
  • OOPS! hook WATCH service unavailable
    OOPS! something went wrong. There was an unexpected error.
  • FOOPS! hook GOOD assessed
    FOOPS! assessment completed in file mode. Accessible checks may remain unassessed.

Publication Assets

Publication asset rows show whether the files required for public release were actually built in the current run.

  • HTML reference page GOOD ready
    Generated public documentation page.
  • Machine-readable source GOOD ready
    Primary Turtle serialization for the release candidate.
  • JSON-LD source GOOD ready
    JSON-LD serialization for the release candidate.
  • Alignment mappings GOOD ready
    Conservative alignment output for review and publication.
  • Examples module GOOD ready
    Separated example and data-like instances.
  • Release bundle GOOD ready
    Bundle status depends on whether the current command executed the release-bundle stage.
  • w3id artifacts GOOD ready
    Redirect templates and publication notes for stable public identifiers.
