Nested ODK Import Flow

Imports are refreshed through the nested ODK workbench and then surfaced through the AIMWORKS release pipeline.

  1. Open ontology_release/odk/src/ontology/.
  2. Run run.bat make refresh-imports.
  3. Review generated import modules under ontology_release/odk/src/ontology/imports/.
  4. Run the normal AIMWORKS release pipeline so the refreshed import status is copied into output/odk/ and reflected in the site.

Shadow Mode Context

ODK is currently operating in shadow mode. The AIMWORKS pipeline remains authoritative while ODK manages standard machine artefacts and import/QC infrastructure in parallel.

HDO role: HDO is the preferred Helmholtz-community source for data-management, metadata, identifier, digital-object, information-profile, schema, and validation concepts.
