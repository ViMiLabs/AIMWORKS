Modeling Patterns

Recommended patterns for measurement, materials, process, provenance, and release publication in H2KG.

The patterns below are intended to keep H2KG conservative, reviewable, and interoperable with the external ontologies it reuses.

Measurement Event Pattern

Use an H2KG measurement class to relate an experimental activity to instruments, parameters, outputs, and normalization context.

  • Model the measurement process as a class or activity-like concept aligned to EMMO or ECHO where appropriate.
  • Link the measurement to instruments with properties such as `usesInstrument`.
  • Keep raw quantity values and example measurement nodes outside the core schema module.

Material and Process Pattern

Represent catalyst-layer materials, manufacturing steps, and outputs with explicit process input and output relations.

  • Preserve local catalyst-layer specific material concepts in the `h2kg` namespace.
  • Reuse broader process or material semantics from EMMO and ECHO where the match is justified.
  • Keep curated named materials in the controlled vocabulary module rather than the asserted schema.

Provenance and Release Pattern

Publish ontology releases with explicit versioning, citation metadata, and stable machine-readable endpoints.

  • Keep the canonical ontology IRI stable and publish version-specific endpoints separately.
  • Record release metadata, creator and contributor information, and validation status.
  • Publish both asserted and inferred serializations together with FAIR and validation reports.
