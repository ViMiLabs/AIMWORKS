# H2KG FAIR Guidance

This entry explains FAIR release quality for H2KG.

## Findable

H2KG should provide:

- stable ontology IRI
- version IRI
- clear title and description
- citation metadata
- predictable namespace policy

## Accessible

H2KG should provide:

- machine-readable serializations
- human-readable HTML documentation
- GitHub Pages-ready outputs
- w3id redirect templates

## Interoperable

H2KG should provide:

- reuse of EMMO, ECHO, QUDT, ChEBI, PROV-O, and DCTERMS where appropriate
- namespace consistency
- explicit mapping review artifacts
- conservative semantic alignment

## Reusable

H2KG should provide:

- clear license
- provenance metadata
- definitions and comments on schema terms
- examples and documentation
- versioning and release notes

## H2KG FAIR Rule

FAIRness is improved more by stable identifiers, good annotations, clear release structure, and explicit reuse than by aggressive ontology rewrites.
