# Annotation Style Guide

Use these annotation rules for local H2KG schema terms:

- labels should be short, readable, and stable
- definitions should explain what the term means in ontology scope
- comments should explain usage when the definition alone is not enough
- avoid vague generic descriptions that could apply to any ontology

Good definition pattern:

- `Class representing X in the H2KG PEMFC catalyst-layer application ontology.`

Good comment pattern:

- explain when to use the term and how it differs from nearby concepts
